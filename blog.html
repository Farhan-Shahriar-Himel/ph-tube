discuss the scope of var let and const

Var ->

var type variable are function scoped. That means they are accessible from every where through out the entire code.

let ->

let type variable are block scoped. They can only be accessed on their own block.

const ->

const type variable are also block scoped. But they must need the initialization when then declared and after initialized they can not be changed or updated.

Tell us the use cases of null and undefined

undefined ->

undefined means the variable has been declared but not initialized by any value.

We can use it when we do not need to assign a value but we need to declare a variable. And this is the default value of a variable

null -?

null means the absence of any object value.

We can use it when we need to clear the variable but at the same time we need to indicate the absence of the value

What do you mean by REST API?

full form of REST API is Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications. It is used widely in web development
